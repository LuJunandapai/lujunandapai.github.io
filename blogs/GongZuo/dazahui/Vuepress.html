# 目录结构

- .vuepress
    - dist: 打包后的文件 用于部署
    - public: 静态文件 如头像
    - styles: 样式 
        - index.css: 可进行覆盖主题样式
    - config.ts: 配置文件 设置标题 导航栏等
- blogs: 文章 设置标签系列
- docs: 文档 设置导航下拉
- README.md: 主页文件 修改标题 图标

# 部署

只需要将打包好的文件夹下的内容上传到远程仓库 最后使用Pages服务即可

GitHuB 部署

创建仓库: 仓库地址需要跟用户个人空间地址一致 否则需要配置 bean 属性(麻烦)

配置分支 启动即可: 分支不能选错

Gitee 部署

创建仓库: 仓库地址需要跟用户个人空间地址一致 否则需要配置 bean 属性(麻烦)

配置分支 启动即可: 分支不能选错

# Git 命令

参考:Git 基本操作 | 菜鸟教程 (runoob.com)open in new window

git add . - 添加文件到暂存区。
git commit | git commit --no-verify -m '提交备注' - 将暂存区内容添加到仓库中 
    
git pull	下载远程代码并合并
git push	上传远程代码并合并

# 克隆 GitHub 不成功

网路的问题 使用代理

参考: git提交或克隆报错open in new window

解决方案

1.在项目文件夹的命令行窗口执行下面代码，

然后再git commit 或git clone取消git本身的https代理，

使用自己本机的代理，如果没有的话，其实默认还是用git的

//取消http代理
git config --global --unset http.proxy
//取消https代理 
git config --global --unset https.proxy

2.科学上网（vpn） 这样就能提高服务器连接速度，能从根本解决 time out 443问题
